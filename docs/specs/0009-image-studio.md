# 0009 Image Studio

## 背景

不只是开发者，内容、运营、设计和产品同样会频繁处理图片：改尺寸、压缩、裁切、转格式、导出多份素材。很多时候只是临时操作，不值得打开完整设计软件；做成百宝箱里的本地优先工具，可以覆盖高频又轻量的图片处理场景。

## 用户故事

作为需要快速处理图片的用户，我希望直接把图片拖进来后，在本地完成压缩、改尺寸、裁切和格式转换，从而更快得到可上传、可分享的结果。

## 本次范围

- 支持拖拽或选择图片文件
- 支持查看基础信息：尺寸、体积、格式
- 支持缩放、等比压缩、输出 JPEG / PNG / WebP
- 支持简单裁切比例：1:1、4:3、16:9
- 支持导出处理后的图片

## 不做什么

- 不做专业图层编辑
- 不做复杂滤镜和调色
- 不做批量目录处理

## 交互与页面

- 左侧配置区：上传、尺寸、质量、格式、裁切比例
- 右侧预览区：原图 / 处理结果对比
- 底部显示输出信息与下载按钮

## 数据与状态

- 输入来源：用户本地图片文件
- 输出结果：浏览器本地处理后的图片
- 是否需要本地存储：不需要持久化原图，可记住最近一次配置

## 验收标准

- 上传图片后可看到预览和基础信息
- 调整尺寸或质量后结果实时更新
- 导出格式切换正常
- 下载结果文件可直接使用

## 风险与待确认问题

- 大图在浏览器中的内存占用需要控制
- 不同格式的透明背景行为需要提示用户
- 裁切交互如果做得太复杂，会影响首版交付节奏
