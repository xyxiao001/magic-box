# 0007 JWT Studio

## 背景

开发者在联调登录、权限和网关场景时，经常需要临时查看 JWT 的 Header、Payload、过期时间和 Claim 内容。现在线上工具虽然多，但很多场景会担心 token 泄漏；做成百宝箱里的本地优先工具更符合实际使用习惯。

## 用户故事

作为前端、后端或测试同学，我希望把 JWT 粘贴进来后，立刻看到结构化解析结果、时间状态和常见 Claim，从而更快定位鉴权问题，而不用把 token 发送到外部网站。

## 本次范围

- 左侧输入 JWT，右侧实时展示解析结果
- 展示 Header / Payload 两块 JSON
- 提取并展示常见时间字段：`iat`、`nbf`、`exp`
- 显示 token 当前状态：有效、未生效、已过期、格式错误
- 支持复制 Header / Payload / 原始 token
- 提供几个常见 token 模板用于演示

## 不做什么

- 不做签名校验
- 不做私钥、公钥管理
- 不做 JWK 拉取与远程验证

## 交互与页面

- 左侧：输入区、模板区、状态提示
- 右侧：状态摘要、Header JSON、Payload JSON、Claim 列表
- 对格式错误要给出清晰提示，不只显示“失败”

## 数据与状态

- 输入来源：用户粘贴的 token 或模板
- 输出结果：浏览器本地解析后的 JSON 与状态信息
- 是否需要本地存储：建议保存最近一次输入

## 验收标准

- 输入标准 JWT 后可以看到 Header 和 Payload
- 能正确识别已过期、未生效、格式错误等状态
- 复制功能可用
- 不依赖网络即可完成核心解析

## 风险与待确认问题

- 不校验签名时，用户可能误以为“可解析”就等于“可信”
- 某些字段不是标准时间戳，需要在 UI 上避免误导
