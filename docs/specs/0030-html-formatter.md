# 0030 HTML Formatter

## 背景

HTML 在前端开发、内容生产、富文本编辑、邮件模板、SSR/静态页面等场景里非常常见。实际工作中常遇到：

- 从页面/控制台/日志里复制出来的 HTML 只有一行，难以阅读与定位层级
- 从外部系统导出的 HTML 缩进混乱，导致 diff 噪音很大
- 需要把一段 HTML 压缩成单行，便于嵌入 JSON、日志或请求 payload
- 希望快速统一缩进与空白风格，而不是手工整理

这类场景适合做成 local-first 工具：不依赖联网，不上传内容，快速得到可读或可嵌入的结果。

## 用户故事

作为前端开发、内容运营或测试同学，我希望把 HTML 粘贴进来后，一键得到更易读的格式化结果，或反向压缩成单行版本，并可调缩进风格，从而更快排查结构、沟通和复制粘贴。

## 首版范围

- 输入：
  - 支持粘贴 HTML 文本（片段或完整文档）
  - 支持包含内联 CSS/JS 的 HTML（不做语义改写）
- 格式化能力：
  - 自动缩进与换行，提升可读性
  - 可配置缩进大小（空格数）
  - 可选：按需补全 `<!doctype html>`（仅当用户开启开关且输入是完整文档/看起来是文档结构时）
  - 支持压缩为单行 HTML（尽量保守，避免破坏文本内容）
- 结果展示：
  - 原始 HTML 与格式化结果对照
  - 行数、字符数统计
  - 一键复制、下载、结果覆盖输入、分享链接
- 模板：
  - 简单页面骨架
  - 常见 UI 片段（列表/卡片）

## 不做什么（首版不含）

- 不做 HTML 的语义重写（例如自动补全缺失闭合标签、重排属性）
- 不做 HTML 校验/报错定位（仅做格式化失败的轻提示）
- 不做 CSS/JS 的独立格式化（只处理整体 HTML）
- 不做 DOM 解析级别的 XSS 清洗（工具不执行/渲染用户 HTML；若页面需要预览，必须明确隔离策略）

## 交互与页面

- 左侧：
  - HTML 输入区
  - 模板按钮
  - 配置项：
    - 缩进空格数
    - 压缩成单行
    - 可选：补全 doctype
- 右侧：
  - 格式化结果
  - 统计信息
  - 操作按钮：复制、下载、清空、结果覆盖输入、复制分享链接
- 辅助提示：
  - 空输入提示，不报错
  - 格式化失败时给出可理解错误信息（如“输入不是有效的 HTML 片段/文档”）

## 数据与状态

- 输入来源：用户输入、样例模板、分享链接恢复
- 输出结果：格式化后的 HTML 文本
- 本地存储：保存最近一次输入与选项（draft + history）
- 网络：不联网，不上传 HTML 内容

## 验收标准

- 输入任意常见 HTML 片段：
  - 能输出更易读的缩进/换行结果
  - 能根据缩进配置改变输出缩进
- 输入完整 HTML 文档：
  - 在不破坏文档结构的前提下进行格式化
  - 开启“补全 doctype”时可生成 `<!doctype html>`（在合理条件下）
- 开启“压缩成单行”：
  - 输出为单行 HTML（或尽量少行），并保证不崩溃
- 输入为空：
  - 有明确提示，不报错
- 用户可以复制/下载结果，或将结果覆盖回输入区，并可通过分享链接恢复状态

## 风险与待确认问题

- 采用基于 formatter 的策略时，需要明确其对“片段 HTML”的容忍度，避免过度严格导致失败率高
- 单行压缩可能会影响文本节点的空白表现，首版要保持规则保守并说明限制
- 是否需要 HTML 预览：首版默认不做预览（避免 `v-html` 带来的安全与隔离复杂度），后续如要做需明确 sandbox 策略
