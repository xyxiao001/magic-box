# 0003 HTTP Lab

## 背景

很多开发者会频繁做简单接口调试，但不一定想每次都打开完整的 Postman 或 Insomnia。一个轻量化的请求构造器可以覆盖最常见的 GET/POST/JSON 调试场景。

## 用户故事

作为开发者，我希望快速配置 URL、Method、Headers 和 Body，并直接看到响应结果，从而完成接口联调和调试。

## 本次范围

- 支持 GET / POST / PUT / DELETE
- 支持设置请求头
- 支持原始 JSON Body
- 展示响应状态、响应头和响应体
- 提供请求示例模板

## 不做什么

- 不做账号登录同步
- 不做复杂环境变量体系
- 不做批量接口集合管理

## 交互与页面

- 顶部配置 URL 和 Method
- 左侧编辑 Headers / Body
- 右侧展示响应结果
- 支持一键格式化 JSON 响应

## 数据与状态

- 输入来源：用户手输
- 输出结果：浏览器请求返回
- 是否需要本地存储：建议后续加入最近请求

## 验收标准

- 可正常发起基础 HTTP 请求
- 请求头和 JSON Body 可编辑
- 响应区能显示状态码、耗时、响应体
- 错误请求会展示清晰错误信息

## 风险与待确认问题

- 浏览器 CORS 限制会影响部分接口
- 大响应体展示可能需要性能优化
