# 0025 JSON Diff / JSONPath

## 背景

接口联调、配置排查和响应体分析时，开发者常见的两个连续动作是：

- 对比两份 JSON，快速确认字段增删改
- 从大体量 JSON 中按路径提取局部数据

仓库里已经有 `JSON Toolkit`、`Diff Studio`、`HTTP Lab`，但三者之间还缺一个“结构化 JSON 对比 + 路径提取”的中间站。新增 `JSON Diff / JSONPath` 后，可以形成更强的联调链路：`HTTP Lab` 查看响应，`JSON Toolkit` 校验格式，`JSON Diff / JSONPath` 继续做结构差异分析与路径查询。

## 用户故事

作为前端、后端或测试同学，我希望把两段 JSON 放进同一个工具里，既能看到结构化差异，也能通过 JSONPath 快速提取目标字段，从而更快定位接口变更、配置漂移和数据异常。

## 首版范围

- 输入：
  - 支持左右两份 JSON 输入
  - 支持格式化和校验输入
  - 支持从模板快速填充接口响应、配置对象等示例
- JSON Diff：
  - 对象字段级比较
  - 数组基础比较
  - 输出新增、删除、修改、未变化统计
  - 支持切换“原始文本对比”和“结构化差异摘要”
- JSONPath：
  - 支持输入 JSONPath 表达式，如 `$.data.items[0].id`
  - 支持返回匹配值列表
  - 支持显示匹配数量与路径
  - 支持复制查询结果
- 链路联动：
  - 页面文案明确推荐与 `HTTP Lab`、`JSON Toolkit`、`Diff Studio` 联动
  - 支持把格式化结果继续作为查询或对比输入

## 不做什么（首版不含）

- 不做完整 RFC 级别的 JSON Patch 生成
- 不做复杂数组智能匹配策略（如按业务主键自动对齐）
- 不做可视化树形拖拽对比
- 不做跨多个 JSONPath 表达式的批量执行

## 交互与页面

- 左侧：
  - 左 JSON 输入
  - 右 JSON 输入
  - 模板按钮、格式化按钮、清空按钮
- 右侧：
  - tabs：`结构差异` / `文本对比` / `JSONPath`
  - 差异统计卡片
  - 差异明细列表
  - JSONPath 输入框与结果区
- 辅助信息：
  - 当前输入是否合法 JSON
  - 匹配结果数量
  - 相关工具快捷入口

## 数据与状态

- 本地存储：
  - 保存左右输入内容
  - 保存当前 tab
  - 保存最近一次 JSONPath 表达式
- 数据全部本地处理，不上传内容

## 验收标准

- 输入两段合法 JSON：
  - 能输出新增、删除、修改、未变化统计
  - 能列出结构差异摘要
- 输入非法 JSON：
  - 能给出清晰错误提示
- 输入合法 JSONPath：
  - 能返回匹配值列表
  - 能显示匹配数量
- 输入不支持或非法 JSONPath：
  - 能给出明确错误或空结果提示
- 用户可以复制差异结果或 JSONPath 查询结果

## 风险与待确认问题

- JSONPath 首版语法支持范围需要收敛，避免一次性覆盖过多表达式形态
- 数组 diff 如果只按索引比较，可解释性强但不一定最贴近业务语义，需要在文案中说明
- 后续可考虑扩展：
  - JSON Patch 导出
  - JSON Pointer
  - 树形差异视图
  - 多表达式查询面板
