# 0021 WebSocket Lab

## 背景

WebSocket 是实时消息、协作编辑、推送订阅、网关链路等场景的基础能力。联调 WS 时经常需要一个轻量客户端：快速连接、发消息、看日志、模拟心跳、重连与断开。相比打开复杂工具，一个集成在百宝箱里的 WS Lab 更符合日常排障节奏。

## 用户故事

作为前端/后端/测试同学，我希望输入 WS 地址后立刻建立连接、发送消息、查看收发日志，并能配置 headers/子协议/心跳，从而快速验证服务端行为与排查连接问题。

## 首版范围

- 连接能力
  - 输入 `ws://` 或 `wss://` 地址
  - 支持子协议 `Sec-WebSocket-Protocol`（多选或逗号分隔输入）
  - 展示连接状态：connecting / open / closed / error
  - 支持手动连接/断开
- 消息能力
  - 发送文本消息
  - 收发消息日志列表（带时间戳）
  - 支持复制单条日志与清空日志
- 心跳与重连（首版简化）
  - 心跳：可配置间隔，定时发送固定 payload（默认 `ping`）
  - 重连：连接断开后可一键重连（不做复杂自动重连策略）
- 数据格式
  - 输入区域支持 JSON 格式化/校验（可选开关）
  - 收到 JSON 时支持折叠预览（可选）

## 不做什么（首版不含）

- 不做 Socket.IO 协议
- 不做二进制消息编辑器（只展示长度或 base64）
- 不做代理转发与抓包

## 交互与页面

- 左侧：连接配置（URL、protocols、心跳配置）+ 连接按钮
- 右侧：消息输入 + 发送按钮 + 收发日志
- 顶部：状态指示、复制/清空按钮

## 数据与状态

- 本地存储：保存最近一次 URL 与配置项
- 不上传消息内容

## 验收标准

- 能成功连接到可用 WS 服务并显示状态
- 发送文本消息后服务端可收到；收到服务端消息能在日志中看到
- 心跳开关生效（按间隔发送）
- 断开与重连流程清晰
- 非法 URL 或连接失败有明确提示

## 风险与待确认问题

- 浏览器对自定义 header 有限制：无法像 HTTP 那样随意加 header，需要在 spec 中明确
- CORS/证书/混合内容（http 页面连 wss）会导致失败，需要给出可读提示
