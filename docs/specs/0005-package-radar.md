# 0005 Package Radar

## 背景

程序员经常需要快速判断一个 npm 包值不值得用，包括版本是否新、包体是否健康、基础信息是否完整。这个过程如果频繁切到 npm、GitHub、npms.io 会很打断流。

## 用户故事

作为开发者，我希望输入包名后，快速看到版本、描述、维护信息和常见安装命令，从而更快做技术选型或版本确认。

## 本次范围

- 支持按包名查询 npm 包
- 展示基础信息：名称、描述、最新版本、发布时间、license
- 展示常用安装命令：`npm` / `pnpm` / `yarn`
- 支持查看最近几个版本号
- 提供常见官方链接入口：npm 页面、homepage、repository

## 不做什么

- 不做复杂依赖树分析
- 不做完整安全审计报告
- 不做登录后的私有 registry 管理

## 交互与页面

- 左侧输入包名并展示候选结果
- 右侧展示当前包的详情、安装命令和版本信息
- 支持一键复制安装命令

## 数据与状态

- 输入来源：用户手输包名
- 输出结果：浏览器请求 npm registry / 公开元数据接口
- 是否需要本地存储：建议保留最近查询记录

## 验收标准

- 输入包名后可获取并展示基础信息
- 常用安装命令正确生成
- 最近版本列表可见
- 网络失败时有清晰错误提示

## 风险与待确认问题

- 不同 registry 返回字段可能存在差异
- 搜索接口和详情接口是否分开使用需要确认
- 如果后续要显示质量评分，需额外评估数据源稳定性
