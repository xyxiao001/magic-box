# 0008 Diff Studio

## 背景

开发过程中经常需要快速比较两段文本、配置、响应体或 Markdown 内容，确认具体改动点。直接开 IDE 或 git diff 有时太重，尤其是临时对比接口返回、文档片段和复制粘贴的配置内容时，轻量本地 diff 工具会非常顺手。

## 用户故事

作为开发者或测试同学，我希望把新旧两段文本贴进去后，立刻看到逐行差异、变更统计和高亮结果，从而快速确认到底改了什么。

## 本次范围

- 左右输入两段文本并实时对比
- 展示新增、删除、未变化三类行
- 输出基础统计：新增行数、删除行数、未变化行数
- 支持复制左右内容
- 提供几个常见模板：配置变更、接口响应、Markdown 文档

## 不做什么

- 不做字符级最优 diff
- 不做三方合并
- 不做文件系统读写和目录比较

## 交互与页面

- 左侧为原始版本，右侧为新版本
- 结果区展示并排对比或统一摘要
- 对空内容、完全一致、差异过大要给出明确反馈

## 数据与状态

- 输入来源：用户本地粘贴或模板
- 输出结果：浏览器本地计算的差异结果
- 是否需要本地存储：建议保存最近一次对比内容

## 验收标准

- 输入两段文本后可以实时看到行级差异
- 完全一致时显示明确状态
- 差异统计正确
- 不依赖网络即可完成核心能力

## 风险与待确认问题

- LCS 算法在超长文本下可能有性能压力，需要控制首版复杂度
- 行级 diff 对 JSON 这类结构化文本仍然够用，但不等于语义级比较
