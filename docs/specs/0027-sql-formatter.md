# 0027 SQL Formatter

## 背景

SQL 是开发、测试、数据分析等场景的高频刚需内容。无论是排查线上查询、整理脚本、写迁移语句，还是把接口背后的查询逻辑发给同事看，大家都会遇到：

- SQL 复制出来全挤成一行，难以阅读
- 关键字大小写不统一
- 条件、子查询、`JOIN` 层级混乱
- 想快速压缩成单行方便嵌入代码或日志

这类场景非常适合做成 local-first 工具：不依赖联网、不上传语句、即时得到整洁结果。

## 用户故事

作为前端、后端、测试或数据同学，我希望把 SQL 粘贴进来后，立刻得到更易读的格式化结果，或反向压缩成单行版本，并控制关键字风格，从而更快完成排查、沟通和复制粘贴。

## 首版范围

- 输入：
  - 支持粘贴 SQL 文本
  - 支持多段 SQL
  - 支持常见语句：`SELECT`、`INSERT`、`UPDATE`、`DELETE`
- 格式化能力：
  - 自动换行和缩进
  - 关键字统一大写或小写
  - 支持压缩为单行 SQL
  - 保留字符串字面量内容
- 结果展示：
  - 原始 SQL 与格式化结果对照
  - 行数、字符数统计
  - 一键复制格式化结果
- 模板：
  - 简单查询
  - 联表查询
  - 更新语句

## 不做什么（首版不含）

- 不做真实数据库语法校验
- 不做执行 SQL
- 不做数据库方言全量兼容选项
- 不做复杂 AST 级别重写
- 不做 SQL Explain 或执行计划分析

## 交互与页面

- 左侧：
  - SQL 输入区
  - 模板按钮
  - 配置项：
    - 关键字大写 / 小写
    - 缩进大小
    - 单行压缩
- 右侧：
  - 格式化结果
  - 统计信息
  - 操作按钮：复制、清空、结果覆盖输入
- 辅助提示：
  - 当前是“格式化模式”还是“压缩模式”
  - 如果检测到明显不完整语句，给出轻提示

## 数据与状态

- 本地存储：
  - 保存最近一次 SQL 输入
  - 保存格式化选项
- 不联网，不上传 SQL 内容

## 验收标准

- 输入常见 SQL：
  - 能输出可读性更好的格式化结果
  - 能根据配置切换关键字大小写
  - 能压缩成单行 SQL
- 输入多段 SQL：
  - 能保留基本段落边界
- 输入空内容：
  - 有明确提示，不报错
- 用户可以复制结果或将结果覆盖回输入区

## 风险与待确认问题

- 不同数据库方言差异较大，首版需要明确只覆盖“通用 SQL 可读性格式化”
- 如果不引入完整 parser，格式化规则需要足够保守，避免破坏原语义
- 对注释、字符串、函数调用的处理要谨慎，避免误拆分
- 后续可扩展：
  - 方言切换（MySQL / PostgreSQL / SQLite）
  - `CREATE TABLE`、DDL、CTE 的专门优化
  - SQL 压缩与高亮双视图
