# 0011 Cron Planner

## 背景

定时任务表达式在运维、后端、脚本自动化和工作流配置中很常见，但很多人并不能直观看懂一串 cron 到底意味着什么。相比单纯查文档，一个可视化地解释 cron、生成表达式并预览未来触发时间的工具更能减少误配风险。

## 用户故事

作为需要配置定时任务的开发者或运维同学，我希望输入或选择 cron 表达式后，立刻看到人类可读解释和接下来几次触发时间，从而确认配置没有写错。

## 本次范围

- 支持标准 5 段 cron 表达式输入
- 生成自然语言解释
- 展示未来若干次触发时间
- 提供常用模板：每小时、每天、工作日、每周
- 支持反向通过模板/选择器生成 cron

## 不做什么

- 不做秒级和 Quartz 全量兼容
- 不做时区数据库管理
- 不做远程任务执行与注册

## 交互与页面

- 左侧输入或选择表达式
- 右侧展示自然语言解释和未来触发列表
- 错误表达式要给出明确提示

## 数据与状态

- 输入来源：用户输入或模板选择
- 输出结果：本地解释文本和时间列表
- 是否需要本地存储：建议保存最近一次表达式

## 验收标准

- 合法表达式能正确解释并展示未来触发时间
- 非法表达式会显示清晰错误
- 模板可快速生成 cron
- 复制表达式功能可用

## 风险与待确认问题

- cron 方言很多，首版要明确只支持哪一类
- 时间解释必须稳定，避免边界条件下误导用户
- 是否默认按本地时区展示需要在 UI 上标明
