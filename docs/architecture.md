# Architecture

## 核心目标

这个项目先做成一个适合持续开发的前端容器，后续再逐步接入真正的工具能力。

## 当前模块分层

- `src/views/`：页面级视图
- `src/router/`：路由定义
- `src/stores/`：Pinia 状态
- `src/data/`：静态规划数据和展示配置
- `src/lib/`：可测试的纯函数

## 工程原则

- 优先做可验证、可迭代的小功能
- 每个新增模块先写 spec，再开始编码
- 页面展示配置与业务逻辑尽量分离
- 可以先用静态数据验证交互，再接真实 API

## 下一步建议

- 落第一个真实工具模块：时间戳转换或 JSON Toolkit
- 为模块页补充细粒度路由
- 加入本地持久化，保存用户置顶和最近使用记录
